# Evaluation

## Evaluation Philosophy

PM Studio AI evaluates both product-management quality and AI-system quality. The system should not only generate artifacts; it should judge whether the artifacts are useful, coherent, and safe enough to review.

## Evaluation Dimensions

### PM Completeness

Checks whether the workspace includes:

- Product thesis
- Target users
- PRD
- MVP
- KPIs
- Experiments
- Risks
- AI architecture

### AI Architecture Fit

Checks whether:

- RAG recommendation matches grounding needs
- Agent recommendation matches workflow complexity
- Human review matches risk level
- Evaluation metrics are included

### Risk Coverage

Checks whether the product addresses:

- Over-reliance on AI
- Unsupported claims
- Workflow mismatch
- Compliance and fairness risks
- Privacy risks

### MVP Discipline

Checks whether:

- Scope is narrow enough
- High-risk automation is excluded
- First milestone validates a core assumption
- Non-goals are explicit

## AI Quality Metrics

- Evidence citation precision
- Unsupported claim rate
- Criteria extraction accuracy
- Human edit rate
- Recommendation consistency
- Risk flag coverage

## Product Quality Metrics

- Time to first usable recommendation
- Workflow completion rate
- User trust score
- Repeat project creation rate

## Business Metrics

- Activation rate
- Trial-to-paid conversion
- Weekly active teams
- Expansion-ready account signals
